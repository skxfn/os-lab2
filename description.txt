========================================================
  Лабораторная работа №2. Создание потоков
========================================================

1. ЦЕЛЬ РАБОТЫ
   Освоить механизм создания потоков в ОС и параллельное выполнение
   задач внутри одного процесса.

2. СТРУКТУРА ПРОЕКТА
   - main.cpp -- единственный файл, содержит три потока:
     main, min_max и average.

3. КАК ЭТО РАБОТАЕТ

   Поток main:
   a) Запрашивает размерность массива и его элементы с консоли.
   b) Создает два потока: min_max и average.
      - Windows: CreateThread(nullptr, 0, threadFunc, &data, 0, nullptr)
        Возвращает HANDLE потока.
      - POSIX:   pthread_create(&thread, nullptr, threadFunc, &data)
        Возвращает 0 при успехе.
   c) Ожидает завершения обоих потоков:
      - Windows: WaitForSingleObject(hMinMax, INFINITE) и
                 WaitForSingleObject(hAverage, INFINITE).
      - POSIX:   pthread_join(tMinMax, nullptr) и
                 pthread_join(tAverage, nullptr).
   d) Закрывает дескрипторы потоков в обратном порядке создания:
      CloseHandle(hAverage) -> CloseHandle(hMinMax).
   e) Заменяет минимальный и максимальный элементы на среднее значение.
   f) Выводит результат.

   Поток min_max:
   a) Проходит по массиву, сравнивая элементы.
   b) После КАЖДОГО сравнения спит 7 мс (Sleep / usleep).
      Это имитирует длительную обработку данных.
   c) Находит минимальный и максимальный элементы + их индексы.
   d) Выводит результат на консоль.

   Поток average:
   a) Проходит по массиву, суммируя элементы.
   b) После КАЖДОГО суммирования спит 12 мс.
   c) Вычисляет среднее арифметическое: sum / size.
   d) Выводит результат на консоль.

   Передача данных между потоками:
   - Используется общая структура ThreadData, передаваемая по указателю.
   - ThreadData содержит: указатель на массив, размер, а также поля
     для результатов (minValue, maxValue, minIndex, maxIndex, average).
   - Потоки записывают результаты в ThreadData, после чего main
     читает их и модифицирует массив.

4. КЛЮЧЕВЫЕ ФУНКЦИИ WinAPI (для защиты)

   CreateThread(lpSecAttrs, dwStackSize, lpStartAddr, lpParam, dwFlags, lpThreadId)
   -- создает новый поток в текущем процессе.
      lpStartAddr -- функция потока с сигнатурой DWORD WINAPI func(LPVOID).
      lpParam     -- указатель на данные, передаваемые потоку.
      Возвращает HANDLE потока.

   WaitForSingleObject(hHandle, INFINITE)
   -- ожидает завершения потока (объект переходит в сигнальное состояние).

   Sleep(dwMilliseconds)
   -- приостанавливает текущий поток на заданное время.

   CloseHandle(hThread)
   -- освобождает дескриптор потока.

   POSIX-аналоги:
   pthread_create() -- создает поток, аналог CreateThread.
   pthread_join()   -- ожидает завершения потока, аналог WaitForSingleObject.
   usleep()         -- спит заданное количество микросекунд (мс * 1000).

5. ОТЛИЧИЕ ПОТОКОВ ОТ ПРОЦЕССОВ (для защиты)
   - Потоки разделяют адресное пространство процесса (общая память),
     поэтому могут напрямую работать с массивом через указатель.
   - Процессы имеют изолированное адресное пространство и должны
     обмениваться данными через файлы, пайпы, shared memory и т.д.
   - Создание потока дешевле создания процесса.
   - В данной лабе синхронизация не требуется, т.к. потоки работают
     с данными только на чтение (массив не модифицируется потоками).

6. ЗАЧЕМ НУЖЕН SLEEP?
   Sleep имитирует длительные вычисления. Без него потоки завершаются
   почти мгновенно и параллельность не наблюдается. С Sleep видно,
   что min_max и average работают одновременно (вывод перемешивается).

7. ПРИНЦИПЫ, СОБЛЮДЕННЫЕ В КОДЕ
   - Структура ThreadData инициализируется memset перед использованием.
   - Переменные цикла объявлены перед циклом (i = 0).
   - static_cast вместо C-style cast.
   - Проверка ошибок создания потоков.
   - Ресурсы освобождаются в обратном порядке создания.
   - Нет магических констант (kMinMaxSleepMs = 7, kAverageSleepMs = 12).
   - Нет using namespace std.
